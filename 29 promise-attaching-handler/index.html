What are Promises?

The Core Concept

A Promise is a JavaScript object representing the eventual completion (or failure) of an asynchronous operation. Think of it as a placeholder for a value that will be available in the future, similar to a reservation at a restaurant.

Just like a restaurant reservation guarantees you a table at a specific time, a Promise guarantees you a result (or error) when an asynchronous operation completes.

JavaScript 
// Creating a Promise - like making a reservation
const myPromise = new Promise((resolve, reject) => {
    // Asynchronous operation (like waiting for a table)
    const tableReady = true; // Simulating success/failure
    
    setTimeout(() => {
        if (tableReady) {
            resolve("Your table is ready!"); // Fulfill the promise
        } else {
            reject("Sorry, no tables available."); // Reject the promise
        }
    }, 2000); // Wait 2 seconds (like waiting for a table)
});

// Using the Promise - like coming back for your table
console.log("Request made. Waiting for response...");
myPromise
    .then(result => console.log(result)) // When table is ready
    .catch(error => console.error(error)); // When no table is available

How Promises Work

The Promise Lifecycle

Every Promise goes through a specific lifecycle with distinct states. Understanding this lifecycle is crucial for mastering asynchronous JavaScript.

When you create a Promise, it starts in the pending state. Once the asynchronous operation completes, the Promise transitions to either:

  • Fulfilled - The operation completed successfully
  • Rejected - The operation failed

Once a Promise is fulfilled or rejected, it becomes settled and its state cannot change. This is called "immutability" - a Promise can only be settled once.

JavaScript 
// Creating a Promise that resolves after 2 seconds
const timerPromise = new Promise((resolve, reject) => {
    console.log("Promise created (pending state)");
    
    setTimeout(() => {
        // After 2 seconds, resolve the promise
        resolve("Promise resolved after 2 seconds!");
    }, 2000);
});

// Checking the state immediately
console.log(timerPromise); // Promise {  }

// After 2 seconds, the promise will be fulfilled
timerPromise.then(result => {
    console.log(result); // "Promise resolved after 2 seconds!"
    console.log(timerPromise); // Promise { "Promise resolved after 2 seconds!" }
});

Promise Execution Flow

Promises are executed immediately when they are created. The executor function (the callback passed to the Promise constructor) runs synchronously, but any asynchronous operations inside it run in the background.

The .then(), .catch(), and .finally() methods are executed asynchronously when the Promise settles. They are added to the microtask queue and processed after the current synchronous code completes.

Promise States

Understanding Promise States

Every Promise exists in one of three states. Understanding these states is fundamental to working with Promises effectively.

Pending

The initial state. The Promise is waiting for the asynchronous operation to complete. It hasn't been fulfilled or rejected yet.

Fulfilled

The operation completed successfully. The Promise now has a resulting value. Handlers registered with .then() will be called.

Rejected

The operation failed. The Promise has a reason for the failure. Handlers registered with .catch() will be called.

JavaScript 
// Example showing all three states
const statePromise = new Promise((resolve, reject) => {
    console.log("Promise is in PENDING state");
    
    // Simulate an async operation
    setTimeout(() => {
        const success = Math.random() > 0.5; // Random success/failure
        
        if (success) {
            console.log("Promise is moving to FULFILLED state");
            resolve("Operation successful!");
        } else {
            console.log("Promise is moving to REJECTED state");
            reject(new Error("Operation failed!"));
        }
    }, 1500);
});

statePromise
    .then(result => {
        console.log("Promise is FULFILLED with:", result);
    })
    .catch(error => {
        console.log("Promise is REJECTED with:", error.message);
    });

Chaining Promises

The Power of Chaining

Promise chaining allows you to sequence asynchronous operations in a clean, readable way. Each .then() returns a new Promise, enabling you to chain multiple operations together.

This avoids the "callback hell" problem and makes your code more maintainable. Think of it as an assembly line where each step depends on the previous one.

JavaScript 
// Chaining promises example - like an assembly line
fetch('https://api.example.com/users')
    .then(response => {
        console.log("Step 1: Received response");
        return response.json(); // Parse JSON and return a new promise
    })
    .then(users => {
        console.log("Step 2: Got users data");
        console.log("First user:", users[0]);
        // Return a new promise for the next step
        return fetch(`https://api.example.com/posts?userId=${users[0].id}`);
    })
    .then(response => {
        console.log("Step 3: Received posts response");
        return response.json(); // Parse JSON and return a new promise
    })
    .then(posts => {
        console.log("Step 4: Got posts data");
        console.log("First post:", posts[0]);
        return posts; // Final result
    })
    .catch(error => {
        console.error("Error in the chain:", error);
        // A single error handler for the entire chain
    });

How Chaining Works

Each .then() returns a new Promise. This is the key to chaining:

  • If you return a value from a .then(), the next .then() receives that value.
  • If you return a Promise from a .then(), the next .then() waits for that Promise to resolve.
  • If you throw an error in a .then(), the chain jumps to the nearest .catch().

This creates a clean sequence of operations where each step depends on the previous one, making complex asynchronous flows easy to understand.

Attaching Handlers

Promise Handlers

Promises provide three main methods for attaching handlers to respond to Promise settlement:

  • .then() - Handles fulfillment and optionally rejection
  • .catch() - Handles rejection (errors)
  • .finally() - Runs regardless of fulfillment or rejection

These methods allow you to control what happens when a Promise settles, making your code more predictable and easier to debug.

JavaScript 
// Using .then(), .catch(), and .finally()
fetch('https://api.example.com/data')
    .then(response => {
        console.log("First .then() - Checking response");
        if (!response.ok) {
            throw new Error('Network response was not ok');
        }
        return response.json();
    })
    .then(data => {
        console.log("Second .then() - Processing data");
        console.log('Data received:', data);
        // Process data
        return data; // Pass data to next handler
    })
    .catch(error => {
        console.log(".catch() - Handling error");
        console.error('Error:', error);
        // Handle errors
    })
    .finally(() => {
        console.log(".finally() - Cleanup");
        // This runs regardless of success or failure
        console.log('Request completed');
        // Cleanup code (hide loading spinner, etc.)
    });

Handler Execution Order

Understanding when handlers execute is crucial:

  • .then() handlers execute when the Promise is fulfilled
  • .catch() handlers execute when the Promise is rejected
  • .finally() handlers execute when the Promise settles (either fulfilled or rejected)

Handlers are executed asynchronously, even if the Promise is already settled when you attach them. They are added to the microtask queue and processed in order.

Real-World Examples

API Data Fetching

One of the most common uses of Promises is fetching data from APIs. The fetch API returns a Promise that resolves to the response.

JavaScript 
// Fetching user data from an API
function getUserData(userId) {
    return fetch(`https://jsonplaceholder.typicode.com/users/${userId}`)
        .then(response => {
            console.log("Response received");
            if (!response.ok) {
                throw new Error('User not found');
            }
            return response.json();
        })
        .then(user => {
            console.log("User data parsed");
            console.log('User:', user);
            return user;
        })
        .catch(error => {
            console.error("Error fetching user:", error);
            throw error; // Re-throw to handle it later
        });
}

// Usage
console.log("Fetching user data...");
getUserData(1)
    .then(user => {
        console.log('Fetched user:', user.name);
        console.log('User email:', user.email);
    })
    .catch(error => {
        console.error('Failed to fetch user:', error.message);
    });

Parallel Operations

Sometimes you need to run multiple asynchronous operations in parallel. Promise.all() and Promise.race() are perfect for this.

JavaScript 
// Running multiple promises in parallel
const userPromise = fetch('https://jsonplaceholder.typicode.com/users/1')
    .then(res => res.json());
    
const postsPromise = fetch('https://jsonplaceholder.typicode.com/posts?userId=1')
    .then(res => res.json());
    
const todosPromise = fetch('https://jsonplaceholder.typicode.com/todos?userId=1')
    .then(res => res.json());

console.log("Starting parallel requests...");

// Wait for all promises to complete
Promise.all([userPromise, postsPromise, todosPromise])
    .then(([user, posts, todos]) => {
        console.log("All requests completed!");
        console.log('User:', user.name);
        console.log('Posts count:', posts.length);
        console.log('Todos count:', todos.length);
        // All data is available now
    })
    .catch(error => {
        console.error('Error in one of the requests:', error);
    });

// Get the result of the first promise to resolve
console.log("Starting race between promises...");
Promise.race([
    new Promise(resolve => setTimeout(() => resolve('First'), 100)),
    new Promise(resolve => setTimeout(() => resolve('Second'), 200))
]).then(result => {
    console.log('Race winner:', result); // "First"
});

Error Handling Patterns

Proper error handling is crucial in asynchronous code. Here are some common patterns:

JavaScript 
// Pattern 1: Centralized error handling
fetchData()
    .then(processData)
    .then(saveData)
    .catch(handleAllErrors); // Single error handler for the chain

// Pattern 2: Specific error handling
fetchData()
    .then(data => {
        try {
            // Process data that might throw synchronous errors
            return processData(data);
        } catch (error) {
            // Convert synchronous errors to promise rejections
            return Promise.reject(error);
        }
    })
    .then(saveData)
    .catch(error => {
        if (error instanceof NetworkError) {
            // Handle network errors
        } else if (error instanceof DataError) {
            // Handle data processing errors
        } else {
            // Handle all other errors
        }
    });

// Pattern 3: Error recovery
fetchData()
    .catch(error => {
        // Try to recover from the error
        return fetchFallbackData();
    })
    .then(processData)
    .catch(error => {
        // Handle errors from both original and fallback
        console.error("All recovery attempts failed:", error);
    });
